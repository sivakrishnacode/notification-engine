Other applications push notification jobs onto a BullMQ queue (Redis-backed).
This engine consumes those jobs and routes them to the correct AWS/third-party
provider based on the "channel" field in the job payload.

Channels:
  - email      → AWS SES
  - sms        → AWS SNS
  - push       → AWS SNS (mobile push / FCM platform endpoint)
  - whatsapp   → Meta Cloud API (or Twilio WhatsApp fallback)
  - in_app     → persist to PostgreSQL + emit via WebSocket (Socket.io)

PROCESSOR FLOW  (notification.processor.ts)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

For each job:
1. Validate payload with Zod; throw UnrecoverableError if invalid
2. Call preferences.isChannelAllowed(userId, channel)
   → if false: write DeliveryLog status=SKIPPED, return
3. Call rateLimit.allow(userId, channel)
   → if false: write DeliveryLog status=RATE_LIMITED, throw Error (triggers retry)
4. Call templates.render(templateId, channel, data) → { subject, body }
5. Call dispatcher.send(job, rendered) → SendResult
6. Write DeliveryLog status=DELIVERED + providerRef
7. Log success via Pino

On provider error:
- Write DeliveryLog status=FAILED + error message
- Re-throw so BullMQ handles retry / DLQ promotion

PROVIDER IMPLEMENTATION DETAILS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

email (AWS SES):
  SendEmailCommand with HTML + plain text body.
  From: SES_FROM_ADDRESS env var.
  Return MessageId as providerRef.

sms (AWS SNS):
  PublishCommand with PhoneNumber (not topic).
  Message = rendered plain text body.
  Return MessageId as providerRef.

push (AWS SNS):
  PublishCommand with TargetArn = deviceToken.
  Payload: { GCM: JSON.stringify({ notification: { title, body } }) }
  Return MessageId as providerRef.

whatsapp (Meta Cloud API):
  POST https://graph.facebook.com/v19.0/{META_PHONE_NUMBER_ID}/messages
  Bearer META_ACCESS_TOKEN
  Body: { messaging_product: "whatsapp", to: waId, type: "text", text: { body } }
  Return response.messages[0].id as providerRef.

in_app:
  Insert DeliveryLog row via PrismaService.
  Emit Socket.io event "notification" to room userId.
  Return DB record id as providerRef.
